# Examples Notes

Use these notes as comparative patterns, not rigid rules.

## Navbar Pattern Options

Minimal pattern:

- `Home | Reference | Articles | News | GitHub`

Best when:

- package is early stage
- article count is low

Grouped pattern:

- `Get started | Reference | Articles (grouped menu) | News | GitHub`

Best when:

- multiple article audiences exist
- workflows need thematic grouping

## Article Naming Conventions

Prefer names that communicate user intent:

- `get-started`
- `workflow-cfa`
- `workflow-dif`
- `faq`
- `migration-from-readme`

Avoid generic names:

- `article1`
- `notes`
- `misc`

## Home Page Title Style

Use concise and stable titles:

- "PackageName"
- "PackageName: Fast measurement tools"

Avoid over-specific release-bound marketing titles.

## README And Home Balance

README should answer quickly:

- What is this package
- How to install
- One minimal example
- Where to go next

Home should explain structure:

- major modules
- maturity and roadmap summary
- links to deeper docs

## Navigation Growth Rule

Add navbar complexity only when users can no longer find content in two clicks.
